# Teachme: file-transclusion sync model (ct-39277)

## 1. The problem
- [ ] Why sync exists at all: Convex/Next.js web app cannot read files on Jason's Mac
- [ ] What the doc-mirror workaround was, and its costs (drift, duplicate source of truth, manual mirroring)
- [ ] Why the branch-vs-main diff is the review artifact (memo worktree flow)

## 2. The solution (proposed v1)
- [x] Push-based, reference-scoped sync: only files named in `![[file:...]]` embeds get synced, at post time
- [x] What's stored per synced file: BOTH full contents — branch version AND main (base) version — keyed by (repo, path, branch)
- [x] Diff is computed at render time in the web client (existing `computeDiff` LCS in DiffView.tsx), not stored
- [x] Why full both-sides content (not just hunks): expand-up/down needs arbitrary context client-side (quizzed ✓)
- [ ] New-file case: no main-side content → whole file is one "added" hunk; render as plain full body, not green diff
- [ ] Unchanged-file case: branch content == main content → render plain body, no diff chrome
- [x] Staleness semantics: snapshot-at-push is deliberate (stable review artifact), not a bug; re-push updates (quizzed ✓)
- [ ] The base is ALSO a snapshot: "diff vs main" means main-at-push-time, not live main
- [ ] What is NOT synced: everything else in the repo — it's not a mirror
