<project_specification>
  <project_name>code-chat-sync - Coding Agent History Sync</project_name>

  <overview>
    A utility that watches Claude Code, Codex CLI, and Cursor history files on developers' machines
    and pushes conversations to a shared Convex database in real-time. Team members install a
    background daemon that monitors local JSONL/SQLite history files and syncs new conversations
    as they happen. A web dashboard allows browsing and searching all team conversations.
  </overview>

  <goals>
    - Enable teams to share coding agent conversations automatically
    - Provide real-time sync as conversations happen
    - Support personal (private) and shared (team) conversation views
    - Zero manual effort after initial setup - runs as background daemon
    - Make it easy to search and learn from team's agent interactions
  </goals>

      <cli_framework>Commander.js for CLI commands</cli_framework>

  <core_features>
    <daemon_watcher>
      <installation>
        - Install via npm: `npm install -g code-chat-sync`
        - Run setup: `code-chat-sync setup` to authenticate and configure
        - Start daemon: `code-chat-sync start` (or auto-start on login)
        - Check status: `code-chat-sync status`
        - Stop daemon: `code-chat-sync stop`
      </installation>

      <file_watching>
        - Watch Claude Code history: ~/.claude/projects/*/*.jsonl and ~/.claude/history.jsonl
        - Watch Codex history: ~/.codex/sessions/**/*.jsonl
        - Detect new lines appended to JSONL files
        - Parse each line as JSON and categorize by record type
        - Handle file rotation and new session files
        - Debounce rapid writes to batch sync efficiently
      </file_watching>

      <sync_behavior>
        - On new session detected: create conversation record in Convex
        - On new message: append to existing conversation
        - Include metadata: timestamp, project path, agent type, model used
        - Hash sensitive data (file paths) for privacy
        - Maintain local cache of synced positions to avoid re-syncing
        - Queue failed syncs for retry with exponential backoff
      </sync_behavior>

      <privacy_controls>
        - Default: all conversations sync to team
        - Config file to exclude specific project paths
        - Environment variable to pause syncing: CODE_CHAT_SYNC_PAUSED=1
        - Local command to mark session private: `code-chat-sync private [session-id]`
      </privacy_controls>

      <daemon_management>
        - Auto-start on system login (configurable)
        - Run in background with minimal resource usage
        - Log to ~/.code-chat-sync/daemon.log
        - Health check endpoint on localhost port
        - Graceful shutdown on system sleep/shutdown
      </daemon_management>
    </daemon_watcher>

    <bidirectional_messaging>
      <architecture>
        - Daemon maintains persistent WebSocket connection to Convex
        - Web/mobile clients send messages via Convex mutation
        - Convex broadcasts to daemon via WebSocket
        - Daemon injects message into active Claude Code session stdin
      </architecture>

      <flow>
        1. User types message in web/mobile input
        2. Message stored in Convex with session_id and pending status
        3. WebSocket pushes message to daemon
        4. Daemon locates active Claude Code session (by session_id)
        5. Daemon writes message to session's input pipe
        6. Daemon confirms delivery, updates message status to delivered
        7. Normal sync captures Claude Code's response
      </flow>

      <error_handling>
        - Session not found: show error, allow retry when session active
        - Daemon offline: queue message, deliver on reconnect
        - Delivery timeout: show failed status, allow retry
      </error_handling>
    </bidirectional_messaging>

## Changelog

### 2025-12-24 - Pivot: Team, Mobile, GitHub OAuth, Bidirectional Messaging, Code Review

**Trigger:** User request to improve team support, add mobile app, messaging to Claude Code, GitHub OAuth, and timeline+review features.

**Added:**
- GitHub OAuth as primary authentication method (email/password kept as fallback)
- React Native mobile app (Expo): sessions, conversations, push notifications, team activity
- Bidirectional messaging: send messages from web/mobile to active Claude Code sessions via WebSocket
- Team invite links: shareable URLs (`codecast.sh/join/CODE`) instead of just codes
- Timeline view: unified view of sessions + commits + PRs
- Code Review UI: fast, keyboard-driven review interface for teams (Vim-style navigation)
- Data model additions: commits, pull_requests, pending_messages, reviews, review_comments, github_connections

**Modified:**
- Authentication: GitHub OAuth becomes primary, email/password is fallback
- Team invites: from 6-char codes to shareable invite links
- Conversation view: now includes inline commits and message sending to active sessions
- Mobile app: moved from out_of_scope to core feature

**Bug Fixes:**
- Daemon status display: settings page shows "Not connected"/"Never" incorrectly

**Rationale:**
- Team collaboration is core value prop - needs frictionless invite flow
- Mobile access enables async review and remote interaction with sessions
- Bidirectional messaging unlocks remote control/guidance of Claude Code sessions
- GitHub OAuth: developers already have GitHub, reduces friction
- Timeline + Code Review: connects code changes to AI sessions, enables team review workflow
